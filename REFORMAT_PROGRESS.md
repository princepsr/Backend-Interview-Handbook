## Format Reference

Every topic follows this structure:

### The Idea
- Analogy-first, no prior knowledge assumed
- 2–3 short paragraphs

### How It Works
- Pseudocode for logic (not full real code)
- One real code block only for the must-memorise gotcha
- Inline tradeoffs per concept

### Interview Lens
- Reading guide note at top
- Delivery tip in italics
- Per question:
  - Question type label (Concept Check / Tradeoff / Design Scenario)
  - One-line answer
  - Full speakable answer (all concepts explained inline)
  - Italic delivery note
  - Gotcha follow-up with full answer
- Common Mistakes at end (consequence explained)
- Quick Revision one-liner at end
