## What is This?

The Java Backend Interview Handbook is a structured, depth-first reference built for engineers preparing for backend software engineering interviews at top-tier technology companies. It covers the full spectrum of topics that appear in SDE1 and SDE2 interviews — from core Java internals and Spring Boot architecture to distributed systems design and real-world LLD case studies.

The primary audience is **SDE2-level engineers** (3–7 years of experience) aiming for mid-to-senior backend roles. SDE1 candidates will find Volumes 1–3 a strong foundation, while SDE2 candidates should work through all six volumes with particular emphasis on Volumes 4 and 5. The handbook targets roles at companies including Amazon, Google, Goldman Sachs, Stripe, Atlassian, Salesforce, Netflix, Uber, Confluent, and similar FAANG+, FinTech, and SaaS organizations.

All code examples and explanations are written for **Java 17** (with Java 21 noted where relevant) and the **Spring Boot 3.x** ecosystem. Topics are grounded in production-grade patterns — not toy examples — so that answers translate directly to real interview discussions.

## Question Format

Every topic in this handbook follows a consistent 10-item structure designed to match how interviewers actually ask and follow up on questions:

| # | Section | Purpose |
|---|---|---|
| 1 | **Difficulty / Frequency / Companies** | Calibrates how much time to invest |
| 2 | **Short Answer** | The one-to-two sentence answer you open with |
| 3 | **Deep Explanation** | Full technical breakdown |
| 4 | **Real-World Example** | How this applies in production systems |
| 5 | **Code Example** | Runnable Java 17 / Spring Boot snippet |
| 6 | **Follow-up Questions** | What the interviewer asks next |
| 7 | **Common Mistakes** | Errors candidates typically make |
| 8 | **Interview Traps** | Subtle gotchas and edge cases |
| 9 | **Quick Revision** | One-line summary for last-day review |

This format ensures you can navigate a topic end-to-end in a single focused session, or jump directly to the section you need during revision.
